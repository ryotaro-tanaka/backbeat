# iOSネイティブアプリ 軽量仕様書（開発中止）

## 1. 目的
m4a形式の音楽ファイルを、iOSネイティブアプリ上で快適に再生できる最小機能を定義する。

---

## 2. 対象範囲

### 2.1 対象機能
- m4a音楽ファイルの再生
- 再生中の停止
- バックグラウンド再生
- シークバーによる再生位置の移動

### 2.2 非対象（本仕様外）
- プレイリスト管理
- イコライザ、音質設定
- ストリーミング再生
- 歌詞表示
- 外部デバイス連携（AirPlay詳細制御など）

---

## 3. 想定利用シナリオ
1. ユーザーがアプリを起動する。
2. m4aファイルを選択または読み込み済みリストから選ぶ。
3. 再生ボタンで再生開始する。
4. 停止ボタンで停止する。
5. シークバーを操作して再生位置を任意の時刻へ移動する。
6. アプリをバックグラウンドにしても再生継続する。

---

## 4. 機能仕様

### 4.1 再生
- m4aファイルを1曲単位で再生できること。
- 再生開始時、現在位置（初回は0秒）から再生する。
- 再生中は再生状態（再生中/停止）をUI上で判別できること。

### 4.2 停止
- 停止操作で再生を停止できること。
- 停止後の再生位置保持仕様：
  - 初期実装では「停止時点の位置を保持」し、再度再生時はそこから再開。
  - ※将来的に「停止時に先頭へ戻す」挙動へ変更可能。

### 4.3 バックグラウンド再生
- ホーム遷移や画面ロック後も再生を継続できること。
- 再生中にアプリがバックグラウンドへ移行しても音声が途切れないこと。
- 必要なiOS設定（Background ModesのAudio）を有効化すること。

### 4.4 シークバー操作
- シークバーで現在再生位置と総再生時間を視覚表示すること。
- ドラッグ操作で任意の時刻へシークできること。
- シーク完了後、指定位置から再生が継続されること。

---

## 5. UI要件（最小）
- 画面要素
  - 曲名表示（またはファイル名）
  - 再生ボタン
  - 停止ボタン
  - シークバー
  - 現在時刻 / 総時間の表示（例：01:23 / 04:56）
- 操作性
  - 片手操作しやすいレイアウト
  - ボタンのタップ領域は十分なサイズを確保

---

## 6. 技術方針（案）
- 実装技術：Swift + SwiftUI
- 音声再生：AVFoundation（`AVAudioPlayer`または`AVPlayer`）
- バックグラウンド再生：`AVAudioSession`のカテゴリ設定 + Background Modes

---

## 7. サーバーレス動画取得構成（構想）

### 7.1 想定クラウド
- Google Cloud Run を利用し、LinuxコンテナでAPIと動画取得処理を動作させる。

### 7.2 処理フロー（案）
1. アプリが動画URLをCloud Run APIに送信する。
2. APIはコンテナ内で `yt-dlp` バイナリ（<https://github.com/yt-dlp/yt-dlp>）を実行する。
3. `yt-dlp` により対象動画をコンテナ内ストレージへダウンロードする。
4. APIは「コンテナ内に保存した動画へアクセスするためのURL」をアプリへレスポンスする。
5. アプリは受け取ったURLを使って動画をダウンロードする。
6. アプリ側ダウンロード完了後、サーバー側（コンテナ側）の動画ファイルを削除する。

### 7.3 補足事項（運用上の前提）
- コンテナ内ストレージは一時領域として扱い、長期保存は想定しない。
- 削除トリガーは、以下のいずれかで実現する。
  - アプリから「ダウンロード完了通知API」を呼ぶ。
  - 期限付きで自動クリーンアップする。
- 取得対象URLのバリデーションや、`yt-dlp` 実行時の制限（タイムアウト/サイズ上限）は別途詳細設計で定義する。

---

## 8. 受け入れ条件（軽量）
- [ ] m4aファイルを再生できる。
- [ ] 停止操作で再生停止できる。
- [ ] バックグラウンド遷移後も再生継続できる。
- [ ] シークバー操作で再生位置変更できる。
- [ ] 主要操作でアプリがクラッシュしない。
- [ ] Cloud Run APIに動画URLを送信し、動画取得処理を開始できる。
- [ ] コンテナ内にダウンロードした動画の取得URLをアプリに返却できる。
- [ ] アプリのダウンロード完了後、コンテナ内動画を削除できる。

## 10. 開発中止の記録
- 本件の開発は最終的に中止とする。
- 中止理由：個人用iPhoneアプリを無料で作成する想定だったが、実機インストールしたアプリは無課金アカウントでは7日で利用不可（再署名・再インストールが必要）となるため。
- 継続運用にはApple Developer Program（99 USD/年）での開発者署名が実質必須であり、正式インストールの前提コストを満たせないため、現時点では見送る。
